# Azure Machine Configuration hardening (one setting = one package) ‚Äî How it fits together

This repository contains **one Machine Configuration (DSC) package per setting**


> Scope: Azure VMs only (not Arc-enabled servers).

---

## 1) Conceptual flow (high level)

1. **DSC configuration** (per-setting) defines:
   - how to **Test** compliance
   - how to **Set** (remediate) the setting

2. `New-GuestConfigurationPackage` turns the compiled **MOF** into a **.zip Machine Configuration package**.  
   (Package type `Audit` or `AuditAndSet`.) 

3. You upload the `.zip` to **Azure Storage** and reference it from Azure Policy using **contentUri** + **contentHash**.

4. `New-GuestConfigurationPolicy` generates a **DeployIfNotExists** policy definition that creates a
   `Microsoft.GuestConfiguration/guestConfigurationAssignments` resource on each targeted VM. 

5. When the Guest Configuration assignment exists, the **Machine Configuration extension** on the VM downloads the package and applies it according to `assignmentType`
   (`Audit`, `ApplyAndMonitor`, `ApplyAndAutoCorrect`). 

---

## 2) Prerequisites you must enforce on VMs

Machine Configuration on Azure VMs requires **at least**:

### 2.1 System-assigned managed identity on the VM
This is required for the VM to authenticate to the Machine Configuration service. 

### 2.2 Machine Configuration extension
Publisher: `Microsoft.GuestConfiguration`  
Type: `ConfigurationforWindows`  
Name: `AzurePolicyforWindows` (expected by many built-ins) 

A Microsoft built-in policy exists to deploy the Windows extension at scale (see `385f5831-...`). 

### 2.3 User-assigned managed identity (UAMI) on the VM (for package download)
Your scenario uses **private Storage** (no SAS / no anonymous blob access). Each Guest Configuration assignment references a `contentManagedIdentity` to access the blob. 

---

## 3) Identities and RBAC (recommended model)

You typically need **two** identities:

### 3.1 VM package-download identity (UAMI attached to VMs)
- Create a UAMI (example: `mc-package-reader-uami`)
- Assign it **Storage Blob Data Reader** on the storage account (or container) hosting the packages.  
  This is explicitly required in the machine configuration policy authoring guidance. 

### 3.2 Policy assignment identity (used by DeployIfNotExists remediation)
DeployIfNotExists requires a managed identity on the **policy assignment** so it can create missing resources. 

Minimum permissions vary by scope and what you deploy, but for this repo you generally need:
- permission to deploy `Microsoft.GuestConfiguration/guestConfigurationAssignments` to VMs
- permission to add UAMI to VMs (Managed Identity Operator actions)

## 4) Step-by-step: build + publish packages

### 4.1 Prepare an authoring machine
On a Windows authoring box (or Windows VM) with PowerShell 7:
- Install PowerShell 7 + required modules (per Microsoft authoring guidance). 
- Install:
  ```powershell
  Install-Module GuestConfiguration -Force
> (Azure CLI commands removed ‚Äî handled by Terraform.)
  ```

### 4.2 Build all packages
From repo root:
```powershell
./scripts/build-all.ps1 `
  -PackageType AuditAndSet `
  -PolicyMode ApplyAndAutoCorrect `
  -ContentUriBase "https://<storage>.blob.core.windows.net/<container>" `
  -UserAssignedIdentityResourceId "/subscriptions/<sub>/resourceGroups/<rg>/providers/Microsoft.ManagedIdentity/userAssignedIdentities/<uami>"
```

This produces, per package:
- `./output/zip/<ControlID>/<ControlID>.zip` (default; configurable via OutputPaths)
- `./output/policy/<ControlID>/` (baseline policy output; default) (generated by `New-GuestConfigurationPolicy`)

### 4.3 Upload zips to Storage
Upload all generated zips to the container used in `-ContentUriBase` using AzCopy or Azure CLI.  
(Ensure blob public access is disabled; use identity-based access.)

### 4.4 (Optional) Hydrate the included policy *templates*
If you want to use the **template JSONs** under each package‚Äôs `policy/` directory:
```powershell
./scripts/hydrate-policy-templates.ps1 `
  -ContentUriBase "https://<storage>.blob.core.windows.net/<container>" `
  -UserAssignedIdentityResourceId "/subscriptions/<sub>/resourceGroups/<rg>/providers/Microsoft.ManagedIdentity/userAssignedIdentities/<uami>"
```

## 5) Step-by-step: deploy policies

### 5.1 Create/Import the custom ‚Äúenforce UAMI‚Äù policy definition
Import `policies/enforce_required_uami_to_vms.json` (portal: Policy ‚Üí Definitions ‚Üí + Policy definition ‚Üí import JSON).

This policy is based on Microsoft‚Äôs ‚Äúmodify VM identity‚Äù tutorial and uses `modify` to attach a UAMI. 

> Assign it with **Enforcement mode = DoNotEnforce** (per Microsoft guidance). 

### 5.2 Create/Import the prerequisites initiative
Import `policies/machine_configuration_prereqs_and_uami_initiative.json` and **replace**:
- `__CUSTOM_POLICY_DEFINITION_ID_ENFORCE_UAMI__`
with the resource ID of your custom policy definition created in 5.1.

This initiative reuses Microsoft‚Äôs built-in Guest Configuration prerequisites plus the custom UAMI policy. 

### 5.3 Assign the prerequisites initiative
Assign the initiative at the scope where you want to manage VMs (subscription or RG).
- Provide parameter `userAssignedIdentityResourceId` = your **VM package-download UAMI**.
- On ‚ÄúRemediation‚Äù, create a remediation task to update existing VMs.

### 5.4 Import & assign per-setting policies
For each package, import the generated policy JSON:
- `./output/policy/<ControlID>/` (baseline policy output; default)

Assign it to the same scope as your VMs.
- Use a **policy assignment managed identity** (user-assigned preferred) because DeployIfNotExists needs it for remediation. 
- Create remediation task(s) for existing VMs.

---

## 6) Notes / operational tips

- Prefer starting with **Audit** / **ApplyAndMonitor** first, then move to **ApplyAndAutoCorrect** once validated. 
- Some settings (e.g., SMB1 feature removal, LSA protection) may require a **reboot** to fully take effect.
- Each package includes a `README.md` with:
  - built-in verification commands (`reg query`, `auditpol`, `wevtutil`, `net accounts`, etc.)
  - standalone build instructions
